item: H-SES-1
date: 2026-04-11T17:49:06Z
summary: Keep launch session state in a runtime-dir registry with JSON-over-Unix-socket control
context: The work item required stable session metadata and a versioned local control protocol, but did not prescribe where active session records should live or how watch/session commands should attach.
decision: Store each active launch in a per-session directory under the local runtime directory, persist session metadata as JSON, expose a versioned line-delimited JSON control socket, and return the observation Unix socket via `watch.attach`.
rationale: A runtime-dir registry keeps metadata discoverable for later commands and native frontends while remaining process-local and easy to clean up. JSON-over-Unix-socket matches the existing local-socket model, is easy to test, and avoids introducing extra CLI surface before policy-mutation semantics are defined.
