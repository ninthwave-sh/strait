item: M-OBS-1
date: 2026-04-11T20:35:00Z
summary: Emit a single policy_reloaded runtime event for both reload and replace mutations
context: The work item required auditable runtime mutation events starting with policy_reloaded and tty_resized, but launch already supported both policy.reload and policy.replace control actions.
decision: Emit policy_reloaded for both control-plane policy mutations and distinguish the action with a source field set to reload or replace.
rationale: This keeps the runtime event surface compact while preserving enough detail for downstream session controllers and watch clients to audit the mutation path without introducing a second near-duplicate event type.
