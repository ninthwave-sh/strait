item: H-CSM-4
date: 2026-04-16T07:30:30Z
summary: Persist only supports local file-backed policy sources
context: The work item requires persist to write a durable Cedar rule to the configured policy source and leave future sessions pre-authorized. Strait also supports git-backed policy sources by cloning them into a temporary worktree for live reload.
decision: Reject persist for git-backed policy sources and require [policy].file for durable updates.
rationale: Writing into the temporary git clone would only affect the current process and would not survive the next launch. Failing fast keeps persist durable, explicit, and reviewable instead of pretending a temporary write is permanent.
