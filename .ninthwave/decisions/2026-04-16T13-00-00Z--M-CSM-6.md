item: M-CSM-6
date: 2026-04-16T13:00:00Z
summary: Ship the first desktop shell as an Electron app and extend SubmitDecision with bounded TTL support
context: The work item recommended Tauri but left the framework open. The shell also needed host-batched blocked-request alerts, tray presence, and a real custom-TTL decision path, while the existing control protocol only supported once/session/persist/deny.
decision: Build the first desktop shell in Electron so the main process can own tray UX and gRPC streams directly, and add a new allow_ttl action plus hold-timer fields to the control protocol.
rationale: Electron let the desktop shell stay thin without adding a second Rust desktop runtime bootstrap inside this work item, while still giving reliable tray behavior and straightforward gRPC streaming. Extending the control protocol kept custom TTL decisions real instead of cosmetic and made the countdown derive from the runtime's actual hold timeout.
